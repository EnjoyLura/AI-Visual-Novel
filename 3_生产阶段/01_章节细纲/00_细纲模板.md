# 章节细纲模板

版本：v0.4

日期：2026-05-17

本文用于统一章节细纲的编写颗粒度。各章节文件只写本章内容，不重复复制本模板。

## 1. 编写原则

- 细纲只负责“这一场要写什么、为什么要写、写到哪里结束”，不直接写完整对白。
- 每个场景必须能转化为 Ren'Py 的一个或多个 label。
- 共通线细纲优先保证节奏、角色入口和分支变量清晰，不提前写个人线后期情绪。
- 场景之间应有明确承接，避免只是把角色依次摆出来露面。
- 纯空间移动不单独作为场景编号。没有对白且只负责移步、切换空间、表现时间流动的内容，应写在相邻剧情场景末尾的“瞬间转场”中。
- 每位女主初登场都要建立一个可被玩家记住的第一印象。
- 男主悠真在共通线前期保持“温和、细心、被卷入、缺少主动选择”的状态。
- 小春作为家庭轴固定存在，但在其他女主入口中只轻推，不压过当前场景的主角。
- 有立绘角色在画面中出现时，必须在“演出提示”写明使用的立绘类型，如常服、校服、工作服或教师装；只用声音出现的角色需标明不出立绘。

## 2. 场景细纲格式

每个场景按以下格式编写：

- 场景编号：用于后续脚本拆分和排期。
- 场景类型：统一使用“剧情场景”。只要包含角色对白，或承载关键独白、信息揭示、情绪变化，就算剧情场景。
- 场景功能：说明这个场景在章节中的叙事作用。
- 时间地点：明确背景素材需求和时间段。
- 登场角色：区分主要对话角色和仅提及角色。
- 入场状态：场景开始时男主和女主的关系、情绪、信息状态。
- 事件流程：按顺序列出场景内发生的关键动作。
- 对白方向：写对白时需要体现的语气、信息点和关系变化。
- 演出提示：背景、立绘类型、服装差分、表情、音乐、音效、转场、独白等。
- 瞬间转场：如需要，用于写明本场结束到下一场之间的纯空间切换。格式示例：`瞬间转场：体育馆 -> 操场（过渡）-> 二年级教室。`
- 结束状态：场景结束时关系、信息或情绪发生了什么变化。
- 分支变量：是否记录选项、候选、解锁、确认或单身线状态。

## 3. 瞬间转场写法

瞬间转场用于让角色从一个明确空间移动到另一个明确空间，避免剧情像事件卡片跳切。它不是独立场景，不单独编号。

- 只写空间顺序和必要的背景切换，例如“体育馆 -> 操场（过渡）-> 二年级教室”。
- 不写新对白，不增加新的剧情冲突。
- 如需要短独白或情绪变化，应保留为剧情场景，而不是瞬间转场。
- 瞬间转场通常放在上一场的“演出提示”之后、“结束状态”之前。
